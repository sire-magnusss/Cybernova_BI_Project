====================================================
0. CRITICAL DATE RANGE RULE
====================================================

Default date range must be the LAST 7 DAYS, not January to June.

Implement this properly:

If the dataset has a timestamp/date column:
- default_end_date = max date available in dataset
- default_start_date = default_end_date - 6 days
- This gives an inclusive 7-day window.

If using live/current generated data:
- default_end_date = today
- default_start_date = today - 6 days

The UI should show:
Last 7 Days
or
Period: <start date> to <end date>

Do not hardcode Jan 1 to Jun 30.
Do not hardcode Jan 1 to Apr 30.
All KPI cards, charts, maps, tables, live pulse, and exports must respect the selected date range.

Use st.session_state:
- date_start
- date_end
- active_dashboard
- active_tab
- selected_market
- selected_role

When the user clicks Reset Filters:
- reset date range to the last 7 days
- reset country/market to All
- reset service to All
- reset segment to All
- reset visit outcome to All
