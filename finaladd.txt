====================================================
SECTION 2 — PUT ALL GRAPHS INSIDE WHITE CARDS
====================================================

Every chart, graph, funnel, map, and table must sit inside a proper visual card.

Card design:
- white background
- subtle border
- soft shadow
- 18–22px rounded corners
- consistent internal padding
- clean top title area
- chart inside with enough spacing
- small interpretation note at bottom of the card

Card structure:
1. eyebrow label (optional)
2. chart title (business-question style)
3. short subtitle
4. chart itself
5. interpretation note / insight banner

Do NOT let charts float directly on the page background.

Apply this to:
- line charts
- funnels
- maps
- donuts
- scatter plots
- heatmaps
- forecast charts
- live graphs
- statistical tables
- action/evidence tables

====================================================
SECTION 3 — CONSISTENT SPACING, HEIGHTS, AND GRID SYSTEM
====================================================

Fix spacing and visual consistency across the whole app.

Requirements:
- equal card padding
- equal row gaps
- equal column gaps
- consistent section spacing
- consistent chart card heights when cards sit side-by-side
- consistent heading spacing
- remove awkward empty whitespace
- avoid cramped charts
- ensure the layout breathes

Use a clean responsive grid:
- 2-column layout for most analytics cards
- full-width cards for hero summaries, report previews, board summary, large maps, and wide tables
- use consistent min-heights for side-by-side cards

Suggested standard card heights:
- KPI cards: compact and equal
- standard chart cards: 420–500px
- mini live cards: 220–280px
- large full-width strategic cards: 520–620px
- evidence tables: enough height to show meaningful rows without feeling cramped

Make all white cards aligned visually.
